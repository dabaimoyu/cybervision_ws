### 第三阶段：定位根本原因

#### 6. 检查双缓冲区实现

**双缓冲区设计原理**：
- ROS 2 线程（写者）写入 `write_buffer` 指向的缓冲区
- OSG 渲染线程（读者）从 `read_buffer` 指向的缓冲区读取
- 每帧通过 `swapBuffers()` 交换读写缓冲区

**问题代码**（`double_buffer.cpp` 第 44-53 行）：
```cpp
void swapBuffers(DoubleBuffer& buffer) {
    std::lock_guard<std::mutex> lock(buffer.getMutex());
    
    int old_write = buffer.getWriteBuffer();
    int new_write = 1 - old_write;
    
    buffer.setWriteBuffer(new_write);  // ✅ 切换写缓冲区
    buffer.setReadBuffer(new_write);   // ❌ 错误！读写指向同一缓冲区
}
```

**问题分析**：
```
初始状态：write=0 (buffer_A), read=1 (buffer_B)

第 1 帧：
  - ROS 2 回调写入 buffer_A (write=0)
  - 主循环读取 buffer_B (read=1) → 空数据
  - swapBuffers(): write=1, read=1  ❌ 都指向 buffer_B

第 2 帧：
  - ROS 2 回调写入 buffer_B (write=1)
  - 主循环读取 buffer_B (read=1) → 还是空数据（刚被覆盖）
  - swapBuffers(): write=0, read=0  ❌ 都指向 buffer_A

循环往复，主循环永远读不到完整的有效数据
```

## ✅ 解决方案

### 修复代码

**文件**：`src/cybervision_edge/src/render_node/double_buffer.cpp`

**修改前**：
```cpp
void swapBuffers(DoubleBuffer& buffer) {
    std::lock_guard<std::mutex> lock(buffer.getMutex());
    
    int old_write = buffer.getWriteBuffer();
    int new_write = 1 - old_write;
    
    buffer.setWriteBuffer(new_write);
    buffer.setReadBuffer(new_write);  // ❌ 错误
}
```

**修改后**：
```cpp
void swapBuffers(DoubleBuffer& buffer) {
    std::lock_guard<std::mutex> lock(buffer.getMutex());
    
    int old_write = buffer.getWriteBuffer();
    int new_write = 1 - old_write;
    
    // 先切换写缓冲区（下一次写入另一个缓冲区）
    buffer.setWriteBuffer(new_write);
    // 然后切换读缓冲区到刚写完的缓冲区（即原来的写缓冲区）
    buffer.setReadBuffer(old_write);  // ✅ 正确
}
```

### 同步修改

**文件**：`src/cybervision_edge/include/double_buffer.hpp`

添加函数声明：
```cpp
/**
 * @brief 交换双缓冲区（在主循环中调用）
 */
void swapBuffers(DoubleBuffer& buffer);
```

**文件**：`src/cybervision_edge/src/render_node/main.cpp`

在主循环中调用交换：
```cpp
// 主循环：从双缓冲区读取数据并更新渲染
while (rclcpp::ok() && qt_app_) {
    auto frame = double_buffer_->read();
    
    if (frame && !frame->objects.empty()) {
        // 更新 OSG 场景
        osg_renderer_->updateObjects(frame->objects, frame->frame_id);
        
        // 更新 Qt GUI
        QMetaObject::invokeMethod(qt_window_, "updatePerformanceData", ...);
    }
    
    // ✅ 每帧结束后交换缓冲区
    swapBuffers(*double_buffer_);
    
    std::this_thread::sleep_for(std::chrono::milliseconds(16));
}
```

## 📚 经验总结

### 1. 双缓冲区设计要点

**核心原则**：
- 写缓冲区始终指向"下一个要写的缓冲区"
- 读缓冲区始终指向"刚写完的那个缓冲区"
- 交换时两者应该指向**不同**的缓冲区

**正确流程**：
```
帧 N:
  写：buffer_A (write=0)
  读：buffer_B (read=1) ← 上一帧的数据
  交换：write=1, read=0

帧 N+1:
  写：buffer_B (write=1)
  读：buffer_A (read=0) ← 刚写完的帧 N 数据
  交换：write=0, read=1
```

### 2. 调试技巧

**多终端隔离**：
- 推理节点和渲染节点分别在不同终端运行
- 避免日志混淆，便于独立观察每个节点的状态

**关键日志点**：
1. **回调函数入口** - 确认数据是否到达
2. **双缓冲区写入** - 确认数据是否正确存储
3. **主循环读取** - 确认读到的数据是否有效
4. **渲染函数调用** - 确认数据是否被使用

**验证方法**：
- 使用 `ros2 topic info --verbose` 检查 DDS 连接
- 添加计数器日志（如每 10 帧打印一次）
- 对比回调帧 ID 和渲染帧 ID 是否匹配

### 3. 常见错误模式

**错误 1**：读写缓冲区指向相同
```cpp
buffer.setWriteBuffer(new_write);
buffer.setReadBuffer(new_write);  // ❌ 错误
```

**错误 2**：忘记调用交换函数
```cpp
// 主循环中没有 swapBuffers()
while (rclcpp::ok()) {
    auto frame = double_buffer_->read();  // ❌ 永远读同一个缓冲区
    // ... 处理数据
}
```

**错误 3**：交换逻辑顺序错误
```cpp
buffer.setReadBuffer(old_write);
buffer.setWriteBuffer(new_write);  // ❌ 先读后写可能导致竞争
```
